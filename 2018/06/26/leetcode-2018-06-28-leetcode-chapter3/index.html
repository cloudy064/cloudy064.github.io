1. 题目描述

无重复字符的最长子串
给定一个字符串，找出不含有重复字符的最长子串的长度。

2. 示例

给定 "abcabcbb" ，没有重复字符的最长子串是 "abc" ，那么长度就是3。
给定 "bbbbb" ，最长的子串就是 "b" ，长度是1。
给定 "pwwkew" ，最长子串是 "wke" ，长度是3。请注意答案必须是一个子串，"pwke" 是 子序列 而不是子串。

3. 题目解答

以后我打算按照从最一般的解法开始讲解，然后逐步优化，讲解优化思路。尽量以通俗易懂的方式讲解每一行代码是怎么来的。

学习算法，在我看来其实是要找到合适的模型，然后在这个模型的基础上再不断的进行优化的这么一个过程。而优化一部分依赖于模型的一些性质，另外一部分依赖于个人经验。一点拙劣的见解，轻点喷。

3.1 暴力法

3.1.1 思路

  1. 初始化最长子串长度max为0
  2. 我们可以简单枚举出所有的子串
  3. 然后对每一个子串判断是否有重复的元素，如果有则直接舍弃，否则计算出长度n
  4. 判断max和n的大小，取最大值赋值给max
  5. 返回最后的max

3.1.2 算法复杂度

时间复杂度： O(n^2)
空间复杂度： O(n^2)

3.1.3 代码实现

3.2 第一次优化

我们很容易可以得到这样一个结论：
假设sub是str的子串，那么如果sub中含有相同的字符，那么str中也一定含有相同的字符。

我们用str[i...j]表示在str上，下标范围是[i, j]的子串。

基于这样一个结论，当我们在暴力法中判断到str[i...j]不满足条件后，对于任意的k>j，str[i...j]也一定不满足条件，可以直接continue到下一个i了。

所以只需要把暴力法中第8行的continue改成break即可

3.3 第二次优化

我们还可以对j进行优化。

如果我们在某一次循环中已经得到，最长的子串长度是n，那么其实接下来我们完全没有必要再去看长度小于n的子串了，因为它再长也不可能大于n。

所以第二轮优化仅仅将第二个for循环中的j的初始值修改为j = i + n即可

3.4 第三次优化

3.4.1 思路

优化了j，我们再稍加思考一下如果优化i。

我们也可以很容易得到这样一个结论；
假设sub是str的子串，那么如果str中不含有相同的字符，那么sub中也一定不含有相同的字符。

现在我们来看，如果str[i...j]不含有相同的字符，而str[i...j+1]含有相同字符，那么是不是就是因为str[j+1]这个字符出现在了str[i...j]中，假定这个字符叫做ch。

所以我们要做的是，获取到下一个i，使得str[i...j]不含有ch这个字符，换一个说法就是我们需要取str[i...j]中ch的下一个字符，从这个字符开始计算下一个符合条件的子串。

3.1.2 算法复杂度

时间复杂度： O(n^2)
空间复杂度： O(n^2)

3.4.3 代码实现

3.5 第四次优化

3.5.1 思路

对于第三次优化，我们在找str[i...j]中的字符ch时，仍然从i到j扫描了一遍，很明显，我们之前已经遍历过str[i...j]中的每一个字符了，这里再扫描一遍就是重复劳动。那么我们是不是可以考虑在第一次遍历的时候就记录下每个字符的位置呢？当然可以！加一个map就行了。

既然已经有map来记录字符出现的位置，那么也就不需要每次创建一个set来判断是否有重复字符了，这样也省略了一大笔开销。

3.5.2 算法复杂度

时间复杂度： O(n)
空间复杂度： O(n)
